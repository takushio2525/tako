# tako 公開前セキュリティ・公開可否監査レポート

> - 監査日: 2026-07-06
> - 対象: リポジトリ全体 — **全ブランチ・全履歴 370 コミット** + 現行ワークツリー（v0.2.8、main = dcf53d6 時点）
> - 手法: gitleaks 8 系（`--log-opts=--all` 全履歴スキャン）/ cargo audit / cargo metadata による全依存
>   ライセンス集計（747 クレート）/ 履歴全文 7.7MB へのパターン grep（秘密情報 14 種・個人情報・IP・URL）/
>   外部通信箇所と危険コードの目視レビュー
> - 前回監査: `.agent/plans/security-audit-report.md`（2026-06-22・186 コミット時点）。本監査はその後の
>   約 180 コミット（リモート接続機能・オーケストレーター・docs 拡充を含む）を含めて全体を再検証した
> - 制約: 本監査は調査と報告のみで、コードの修正は行っていない

## 総合判定: 条件付き OK

**API キー・トークン・パスワード・秘密鍵・Webhook URL 等の「真の秘匿情報」は、現行ファイル・全履歴
ともにゼロ**。秘匿情報の観点では public 化してよい。ただし公開前に以下の条件を満たすこと。

| # | 条件 | 位置づけ | Issue |
|---|---|---|---|
| 1 | **ライセンスは GPL-3.0-or-later のまま公開する**。Apache-2.0 への変更は GPL 依存 3 クレートにより法的に不可能 | **必須** | [#75](https://github.com/takushio2525/tako/issues/75) |
| 2 | 個人情報を含む検証アーティファクト（`.playwright-mcp/`・`pwa-*.png`・`.wrangler/cache/`）を削除し .gitignore に追加する | 強く推奨 | [#76](https://github.com/takushio2525/tako/issues/76) [#77](https://github.com/takushio2525/tako/issues/77) |
| 3 | git 履歴に残る個人情報の扱い（受容 or filter-repo 書き換え）を明示的に決める。**書き換えるなら公開前が唯一の機会** | 強く推奨（受容も可） | [#81](https://github.com/takushio2525/tako/issues/81) |
| 4 | リレー worker（無認証・無レートリミット）の公開後悪用リスクへの方針を決める | 推奨 | [#78](https://github.com/takushio2525/tako/issues/78) |

条件 1 を守らない（Apache-2.0 で公開する）場合の判定は **NG**（依存ライセンス違反の状態で公開することになる）。
条件 2〜4 は秘密情報ではないため未対応でも公開自体は可能だが、公開後は不可逆になる項目（条件 3）を含む。

## 発見事項（重大度順）

### H-1. Apache-2.0 への変更は GPL 依存により不可能 → [#75](https://github.com/takushio2525/tako/issues/75)

- 重大度: **High**（公開方針のブロッカー）
- 場所: Cargo.lock 依存ツリー — `zlog 0.1.0` / `ztracing 0.1.0` / `ztracing_macro 0.1.0`（Zed monorepo 由来、GPL-3.0-or-later）
- 内容: GPUI のトランジティブ依存に GPL クレートが 3 つ含まれ、スタティックリンクされるため、配布物全体に
  GPL の義務が及ぶ。Apache-2.0 での配布は依存ライセンス違反。前回監査 C-1 と同じ結論を v0.2.8 の
  依存ツリー（747 クレート）で再確認した。その他 744 クレートはすべて GPL-3.0 互換
  （MIT / Apache-2.0 / BSD / Zlib / Unicode-3.0 / MPL-2.0 ほか。`self_cell` の「Apache-2.0 OR GPL-2.0-only」は
  デュアルで Apache 選択可。license フィールド欠落の `gpui_shared_string` / `gpui_util` は LICENSE-APACHE
  同梱で実質 Apache-2.0）
- 推奨対応: GPL-3.0-or-later のまま公開する。現行宣言（LICENSE / Cargo.toml / AGENTS.md）は一貫しており追加作業なし

### H-2. `.playwright-mcp/` と `pwa-*.png` に個人情報が tracked → [#76](https://github.com/takushio2525/tako/issues/76)

- 重大度: **High**（公開前に削除推奨）
- 場所: `.playwright-mcp/*.yml`（tracked 20 ファイル、2026-06-22 コミット）、`.playwright-mcp/*.png`、
  リポジトリ直下の `pwa-panes-list.png` / `pwa-terminal-view.png`
- 内容: Playwright MCP のページスナップショット（テキスト）と PWA 検証スクショ（画像・目視確認済み）に、
  シェルプロンプト `shiozawatakumi@shiozawaMacPro`（macOS ユーザー名 + ホスト名）と
  `/Users/shiozawatakumi/Library/Application Support/tako/...` のホームパスが含まれる。
  `.playwright-mcp/` は .gitignore 未登録で、現在も untracked の新規ファイルが発生し続けている
- 推奨対応: `git rm` で削除 + `.gitignore` に `.playwright-mcp/` を追加。履歴残存の扱いは #81 で判断

### H-3. `.wrangler/cache/` に Cloudflare account_id + メールアドレスが tracked → [#77](https://github.com/takushio2525/tako/issues/77)

- 重大度: **High**（公開前に削除推奨）
- 場所: `.wrangler/cache/wrangler-account.json`（account id + `"Shiozawataku2525@gmail.com's Account"`）、
  `.wrangler/cache/pages.json`（account_id + project_name）
- 内容: wrangler CLI のローカルキャッシュの誤コミット。account_id 自体は API トークンなしでは操作できず
  秘密ではないが、メールアドレスとアカウントの紐付けを公開する必要はない
- 推奨対応: `git rm -r .wrangler` + `.gitignore` に `.wrangler/` を追加。履歴残存の扱いは #81 で判断

### M-1. リモートリレー worker が無認証・無レートリミット → [#78](https://github.com/takushio2525/tako/issues/78)

- 重大度: **Medium**（公開でリレー URL が既知になり悪用面が拡大）
- 場所: `web/tako-remote-worker/src/index.js`、`crates/tako-control/src/remote.rs:48` /
  `web/tako-remote/src/api.js`（`DEFAULT_RELAY_URL = https://tako-remote-relay.takushio2525.workers.dev`）
- 内容: register / resolve とも認証なしで、machineId（UUID v4）の推測困難性だけが防御。machineId が
  漏れると第三者が PWA の接続先トンネルを差し替え可能（tako 本体側にトークン認証があるため即乗っ取りには
  ならないが、偽画面誘導の入口になる）。レートリミットがなく、公開後は第三者の相乗り・KV 埋め・無料枠消費が
  可能。入力検証（UUID 形式・trycloudflare 限定・TTL 24h）は実装済みで良い
- 推奨対応: 軽い認証（登録シークレット + HMAC）or Cloudflare 側レートリミット or
  「公共ベストエフォート・`TAKO_RELAY_URL` で差し替え可」の明記、のいずれかを公開前に決める

### M-2. quick-xml に RUSTSEC-2026-0194 / 0195（severity 7.5 high・DoS） → [#79](https://github.com/takushio2525/tako/issues/79)

- 重大度: **Medium**（実害は限定的だが公開後に cargo audit で指摘されうる)
- 場所: Cargo.lock — `quick-xml 0.30.0`（`xcb` 経由）/ `0.39.4`（`zbus_xml` 経由）。いずれも GPUI の
  トランジティブ依存で、X11 / D-Bus という Linux ターゲット限定のため macOS / Windows 配布物には
  含まれない見込み
- 内容: メモリ枯渇 DoS と quadratic run time の 2 件（2026-06-29 公表 = 前回監査より後）。解は >= 0.41.0
- 推奨対応: `cargo update` で更新可否を確認、不可なら GPUI 依存更新待ちとして追跡。CI への cargo audit 追加を推奨

### L-1. git 履歴に残る個人情報（履歴書き換えの判断） → [#81](https://github.com/takushio2525/tako/issues/81)

- 重大度: **Low**（秘密ではないが公開後は不可逆）
- 場所: H-2 / H-3 の履歴残存分に加え、履歴のみに (a) 旧リレー URL
  `https://tako-remote-relay.shiozawa-takumi.workers.dev`（**実名入り**サブドメイン。現行コードは
  takushio2525 版に変更済み）、(b) コミット author メール `shiozawataku2525@gmail.com`（338 コミット中 296 件）
- 内容: public 化で全ブランチ・全履歴が公開され、削除済みブランチも PR refs 経由で参照可能になりうる。
  GitHub は一度 public 化したコミットをフォーク経由で保持しうるため、書き換えは公開前が唯一の機会。
  なお実名は git author 名（TakumiShiozawa）としても全コミットに含まれるため、消しても実名自体は公開される
- 推奨対応: 「受容（作業ゼロ・SHA 維持。推奨）」か「filter-repo で該当パス削除 + mailmap で noreply 化 +
  force push」を明示的に決める。旧 worker が Cloudflare 上で稼働中なら削除（リポ外作業）

### L-2. FileOp::Trash の AppleScript エスケープが不完全 → [#80](https://github.com/takushio2525/tako/issues/80)

- 重大度: **Low**（前回監査 W-3 の持ち越し。実害は限定的）
- 場所: `crates/tako-control/src/dispatch.rs:1186-1189`
- 内容: `"` と `\` のみエスケープして `osascript -e` へ文字列連結。改行を含むパス（macOS では合法）で
  AppleScript リテラルを壊せる余地。呼び出せる主体は既にローカル実行能力を持つため権限昇格ではないが、
  公開後に「インジェクション」と指摘されやすい
- 推奨対応: 改行・制御文字を拒否、または `on run argv` の引数渡しに変更

## 確認して問題なしだった項目

### 1. 秘匿情報（現行 + 全履歴）— 問題なし ✅

- **gitleaks（全ブランチ 370 コミット）: 検出 3 件はすべて false positive**
  （`remote.rs` の `#[test]` 内 QR 生成テスト用ダミー値 `abc123def456` + テスト用プライベート IP）
- 手動パターン grep（履歴全文 7.7MB）: GitHub PAT（`ghp_`/`github_pat_`/`gho_`）、Anthropic（`sk-ant-`）、
  AWS（`AKIA`）、Slack（`xox?-`、webhook）、Discord webhook、JWT（`eyJ...`）、秘密鍵ブロック
  （`BEGIN PRIVATE KEY`）、`Authorization: Bearer/Basic` 実値、`CF_API_TOKEN`/`CLOUDFLARE_API`、
  ngrok — **すべてゼロ**
- `.env*` / `*.key` / `*.pem` / `*.p12` / `.dev.vars`: 履歴上に一度もコミットされていない
  （履歴に登場した全 270 パスを列挙して確認）
- `TAKO_TOKEN`: 実行時 CSPRNG 生成のみ。ハードコードなし・ログ出力（println/eprintln/log 系）なし
- CI（`.github/workflows/ci.yml`）: secrets 参照なし
- `wrangler.toml` の KV namespace ID: コミットされているが、API トークンなしでは操作不能な識別子であり
  Cloudflare 公式もコミット前提。問題なし
- trycloudflare.com の言及 61 件: すべて `foo-bar-baz` 等のダミー。実トンネル URL の漏れなし

### 2. 個人情報 — H-2 / H-3 / L-1 以外は問題なし ✅

- グローバル IP・内部サーバー URL / IP: **ゼロ**（履歴全文から IP 形式を抽出しプライベート帯・例示を除外して確認）
- ソースコード・ドキュメント内のメールアドレス・実名: なし（`/Users/foo` 等のダミーのみ。
  `takushio2525` / `dev.takushio.tako` は公開リポのオーナー識別子として意図的）
- docs サイトの画像: 自作 SVG 3 点のみ。履歴から削除済みの `shots/*.png` も取り出して目視し、
  docs スクショ・マスコット画像のみで個人情報なし
- `.agent/` の作業ログ: 個人情報・他プロジェクトへの言及なし。ただし前回監査レポート
  （`.agent/plans/security-audit-report.md`）と本レポート自体が弱点の指摘（L-2 等）を含んだまま
  公開される点は認識しておく（記載済みの弱点はいずれも軽微で、公開しても悪用価値は低い）

### 3. 危険・不適切なコード — 問題なし ✅

- **外部への通信は 4 経路のみで、いずれも意図された機能**。意図しないデータ送信・テレメトリなし:
  | 経路 | 送信先 | 送信内容 | 契機 |
  |---|---|---|---|
  | 更新チェッカー（`update_checker.rs`） | GitHub Releases API | リクエストのみ（ユーザーデータなし） | 起動時チェック |
  | リレー登録（`remote.rs`） | 作者の Workers（M-1） | machineId + トンネル URL | リモート機能 opt-in 時のみ |
  | AI 自動リネーム（`autorename.rs`） | ローカルの claude CLI 経由で Anthropic | ペイン末尾テキスト・cwd 等 | 機能 ON 時（`tako autorename` で OFF 可） |
  | Web ビュー（`webview.rs`） | localhost の Chrome CDP のみ | — | — |
- コマンド実行: tmux / git / claude / osascript 等はすべて `Command::new` + 個別 `arg()` でシェル非経由。
  シェルインジェクション経路なし（L-2 の AppleScript 文字列連結のみ要改善）
- ネットワーク待受: MCP HTTP は `127.0.0.1` バインド + Bearer トークン + Origin 検証
  （DNS リバインディング対策）。IPC は Unix ソケット 0600 + トークン。リモート機能は opt-in +
  トンネル + トークン認証
- `eval` / `curl | sh` 的パターン: なし
- `design-ref/`: 自作（Claude Design 出力）で他者著作物なし

### 4. 公開準備 — ライセンス以外は問題なし ✅

- LICENSE（GPLv3 本文）・`Cargo.toml`・AGENTS.md の宣言は GPL-3.0-or-later で一貫（H-1 の結論と整合）
- MPL-2.0 依存 3 件（cbindgen / dwrote / option-ext）: ファイルレベル copyleft で GPL-3.0 互換
- README / docs / CHANGELOG: private 前提の記述なし（公開を前提とした書き方になっている）
- cargo audit: M-2 以外は unmaintained 警告のみ（async-std / bincode / instant / paste /
  proc-macro-error2 — すべて GPUI 経由で tako 側で対処不可、前回監査から変化なし）
- Issue テンプレート・CI（macOS / Windows ビルド）は公開に耐える状態

## 公開作業時の注意（判定条件の補足）

1. **順序**: #75（ライセンス方針の確定）→ #76 / #77（ファイル削除）→ #81（履歴の扱い決定。書き換えるなら
   このタイミングで filter-repo + force push）→ public 化。公開後の履歴書き換えはフォーク・キャッシュに
   残るため実質不可能と考えること
2. public 化すると origin の全ブランチ（feature/* 等 20 本以上）と削除済みブランチの PR refs も見える。
   非公開にしたい実験ブランチがあれば公開前に origin から削除する（今回の監査では全ブランチを走査済みで、
   秘匿情報は存在しない）
3. 本レポートと前回監査レポートも公開される。記載済みの弱点（L-2 等）はいずれも軽微だが、
   公開範囲に含めたくない場合はコミット前に除外を判断する
