# .agent/ — AI 向け詳細仕様の置き場

このディレクトリは **AI（Claude Code 等）が実装中に参照する詳細仕様** を置く場所。
人間向けの説明は `../README.md` を参照。

## ファイル

### 静的な仕様（プロジェクトの不変部分）

- `concept.md` — コンセプト・競合分析・差別化軸・Non-goals（最上位。迷ったらここに戻る）
- `requirements.md` — 機能要件（FR 番号 + MoSCoW 優先度）・非機能要件
- `architecture.md` — 技術スタックとリスク、レイヤ構成、ドメインモデル、3 層制御プレーン
- `roadmap.md` — フェーズ計画と各フェーズの Exit Criteria
- `conventions.md` — 命名・エラー・ログ規約（コード着手後に拡充）

公開 API（CLI サブコマンド・MCP ツール）のシグネチャは現状 `architecture.md` に
集約している。コード着手後にインターフェースが固まったら `api.md` として分離する。

### 動的な作業文脈（毎ターン読む・更新する）

- `activeContext.md` — **現在の対象・直近の観点・次の一手**。AI が**毎ターン上書き**するスナップショット
- `progress.md` — **完了タスクの時系列**。AI が作業完了時に**末尾へ追記**。肥大化したら archive 提案

## 書き方の原則

- AI が探す語をタイトル・見出しに含める
- 説明調ではなく**列挙的・簡潔**に書く
- 1 ファイル 100〜300 行を目安に。長くなったら章ごと分割
- 親リポジトリの `AGENTS.md` から参照される前提（重い設計書はバックティック参照）
