# manual-checks.md — 手動確認チェックリスト

> セルフテスト（`TAKO_SELF_TEST=1`）で機械検証できない、人間の目と実際の OS 統合が
> 必要な確認項目。リリース前・該当機能の変更時に通す。

## IME 変換中表示（FR-1.9、Phase 3.5）

セルフテスト 37〜39 は EntityInputHandler の状態遷移（保持・確定・unmark）のみを検証する。
実際の IME（NSTextInputClient）経由の見た目・候補ウィンドウは以下を手動確認する。

前提: macOS の日本語入力（ライブ変換 OFF 推奨で確認が安定）。`cargo run -p tako-app` で起動。

- [ ] 日本語 IME で「nihongo」と打つ → カーソル位置に「にほんご」が**下線付き**で
      インライン表示される（確定前。シェルには何も送られていない）
- [ ] スペースで変換 → 変換候補に置き換わり、**注目文節が太い下線 + 選択色**で強調される
- [ ] 変換候補ウィンドウが**変換中テキストのすぐ下**に出る（画面左上や別ペインに出ない）
- [ ] enter で確定 → 確定文字列が**1 回だけ**シェルへ入る（二重入力しない）
- [ ] esc でキャンセル → 未確定文字列が消え、シェルには何も送られない
- [ ] 複文節（例:「きょうはいいてんき」）でカーソルキーで文節を移動 → 太下線が文節に追従する
- [ ] 変換中にウィンドウをリサイズしても表示位置がカーソルに追従する
- [ ] ペイン分割後、**フォーカス中のペイン**のカーソル位置に変換中テキストが出る
- [ ] 英数モードでの直打ち（ASCII）が二重入力しない（stop_propagation の回帰確認）
- [ ] dead key（ABC 配列で option-e → e = é など）が機能する
- [ ] cmd-d / cmd-t 等のショートカットが IME 有効時（ひらがなモード）でも効く

既知の制約（実装メモ。`crates/tako-app/src/main.rs` の EntityInputHandler 実装コメント参照）:

- 変換中テキストはペイン右端で折り返さない（長い未確定文字列はペイン外へはみ出す）。
  日常入力では文節確定が先に来るため許容。問題になれば折り返し対応を検討
- スクロールバック表示中（カーソル非表示）は変換中テキストのアンカーが取れないため
  オーバーレイ・候補ウィンドウの位置出しを行わない（変換自体は可能で確定は届く）

## ペイン境界ドラッグリサイズ（Phase 1 残骨格）

座標→比率換算とツリー更新はセルフテスト 5b で機械検証済み。カーソル形状と手触りは目視で確認:

- [ ] cmd-d で左右分割 → 境界線にマウスを近づけると**左右リサイズカーソル**（⇿）になる
- [ ] cmd-shift-d で上下分割 → 境界線で**上下リサイズカーソル**（↕）になる
- [ ] 境界をドラッグ → ペイン比率がマウスに追従して滑らかに変わり、PTY も即リフロー（cols/rows 追従）
- [ ] ネストした分割（例: 右ペインをさらに上下分割）で**内側の境界だけ**が動く（外側は無傷）
- [ ] 端まで引いても潰れない（MIN_SHARE/MAX_SHARE=0.1/0.9 でクランプ）
- [ ] 境界ハンドル上でのドラッグは選択（copy-on-select）を誤発火しない
- [ ] ドラッグ中にウィンドウ外でマウスを離した後、ペイン内の通常クリック・ドラッグで
      リサイズが誤発火しない（カーソル形状が変わる領域とドラッグ開始領域が常に一致する。
      機械検証はセルフテスト 5c）

## シェル統合（FR-2.4.1、Phase 4）

zsh の自動注入（ZDOTDIR 方式）はセルフテスト 41/41b で機械検証済み。
bash / fish は注入経路（PROMPT_COMMAND / XDG_DATA_DIRS）が異なるため手動確認する:

- [ ] ペイン内で `bash` を起動 → `cd /tmp` → `tako list` の該当ペインに cwd が出る
- [ ] 同 bash で `false` → `tako list` の state が failed / exit_code 1
- [ ] ペイン内で `fish` を起動（インストール済みなら）→ 同上
- [ ] p10k / starship 等のプロンプトカスタマイズと共存できる（プロンプト崩れなし）
- [ ] `TAKO_NO_SHELL_INTEGRATION=1 tako-app` で統合が無効化される（cwd / state が unknown のまま）

状態インジケータ（FR-2.1.4。集約ロジックはユニットテスト済み、見た目を目視）:

- [ ] `sleep 5` 実行中、ペイン右上にアクセント色ドット + タブにも同色ドットが出る
- [ ] `false` 実行後、赤ドットに変わり、次のコマンド実行開始まで保持される
- [ ] 非アクティブタブのペインが失敗しても、そのタブに赤ドットが出る（タブ横断の把握）
- [ ] プロンプトに戻って正常終了（exit 0）したらドットが消える

## tmuxview タブ（FR-2.13）

コマンド層・状態遷移はセルフテスト 48/49 で機械検証済み。見た目と操作感を目視で確認:

- [ ] タブバー右端に「tmux」タブが固定表示され、× が無い（閉じられない）
- [ ] tmux セッションがあるとき、名前 / attached・detached / 作成時刻 / window 一覧が出る
- [ ] tako 内のペインで `tmux attach` したセッションに「tako タブ N / ペイン M」が出る
- [ ] 外部ターミナル（iTerm2 等）で attach したセッションは「tako 外」と出る
- [ ] kill ボタン → インライン確認 →「kill する」で消える、「やめる」で何も起きない
- [ ] 表示中に外部で tmux を作る / 殺すと 2 秒以内に一覧へ反映される
- [ ] 通常タブのクリック・cmd+数字・cmd+T で tmuxview から抜けられる

## role / title バッジ表示（FR-2.1.3、Phase 3 残）

role / title の設定・取得自体はセルフテスト（`tako title / role 設定`）で機械検証済み。
バッジの見た目は目視で確認:

- [ ] `tako title --role worker-1 REVIEWER` → ペイン右上に「REVIEWER · worker-1」バッジが出る
- [ ] role のみ / title のみ設定でもバッジが出る（両方未設定なら出ない）
- [ ] バッジがターミナル 1 行目の文字と重なっても読める（タブバー背景色の下地）
- [ ] バッジ上のクリックでもペインフォーカス・選択が通常どおり動く

## .app バンドル（Phase 3.5）

`scripts/build-app.sh --verify` の機械検証（バンドル版バイナリでのセルフテスト 39 項目）に
加えて以下を手動確認する。

- [ ] `dist/tako.app` をダブルクリック（または `open dist/tako.app`）で起動する

## AI 自動リネーム（FR-2.12）

トグル・rename API・手動優先・ヒューリスティック適用はセルフテスト（項目 50〜52）で
機械検証済み。claude CLI を実際に呼ぶ経路と見た目は常用で確認:

- [ ] コマンド実行が一段落して数秒後（静穏 4 秒 + クールダウン 30 秒）、タブ名・ペイン名が
      作業内容に沿った短い日本語名へ変わる
- [ ] `tako title` / `tako tab rename` で手動命名したタブ・ペインは自動で上書きされない
- [ ] `tako tab rename --tab N ''`（空文字）で手動指定が解除され、自動リネームが再開する
- [ ] `tako autorename off` で止まり、再起動後も OFF が維持される（settings.json）
- [ ] claude CLI の無い環境（PATH から外す）で OSC タイトル / cwd 由来の名前に劣化する

## listen ポート検知と提案チップ（FR-2.4.2〜2.4.4）

検知・チップの状態遷移・OFF トグルはセルフテスト（項目 53〜55）で機械検証済み。
見た目と実サーバーでの体験は常用で確認:

- [ ] `npm run dev` 等で dev サーバーを起動すると数秒以内にペイン下端へ
      「localhost:PORT（プロセス名）が listen 中」チップが出る
- [ ] 「ブラウザで開く」で既定ブラウザに該当 URL が開き、チップが消える
- [ ] 「×」で消え、同じサーバーが動いている間は再提案されない（再起動で再提案）
- [ ] `tako portdetect off` でチップ・list の listen_ports が消え、再起動後も OFF が維持される
