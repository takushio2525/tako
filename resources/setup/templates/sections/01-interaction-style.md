# 対話スタイル

## 必須概念

- 応答の簡潔さ・詳細さの度合いを定義する
- 確認を求めるタイミングのルールを定義する
- 不明点があったときの行動を定義する

## 参考テンプレート

### 初心者向け

```markdown
## 対話スタイル

- コードの変更理由を毎回簡潔に説明する
- 専門用語を使う場合は簡単な補足を添える
- 大きな変更の前にプランを提示し、確認を求める
- エラーが出たら原因と対処法をわかりやすく説明する
```

### 中級者向け

```markdown
## 対話スタイル

- 要点から先に出す。冗長な前置きは不要
- 曖昧な指示には最も妥当な解釈を一文で宣言してから着手する
- 既存コードへの波及・より素直な実現方法があれば着手前に指摘する
- 大きな変更の前にプランを提示し、確認を求める
```

### 上級者向け

```markdown
## 対話スタイル

- 要点から先に出す。社交辞令や末尾の要約は不要
- 短い指示は文脈から最も妥当な解釈で着手する（質問返しで止まらない）
- 前提のズレや落とし穴に気づいたら黙って従わず先に指摘する
- 「言われた作業をやる」のではなく「なぜそれが必要か」を理解し、
  より良い方法があれば先に提案する
```
