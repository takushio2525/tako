## 必須概念

- 依頼されていない変更を勝手に行わないルールを定義する
- コメントの方針を定義する

## 参考テンプレート

### 共通（全レベル）

```markdown
## コード品質

- 依頼された範囲だけを変更する。関係ない箇所のリファクタや機能追加はしない
- 新しいライブラリを追加する前に、既存の依存で実現できないか確認する
- セキュリティ脆弱性（インジェクション、XSS 等）を作り込まない
```

### コメント方針（簡潔）

```markdown
## コメント

- コメントは基本不要。WHY が非自明な場合のみ書く
- コードが何をしているかの説明は書かない（コード自体で表現する）
```

### コメント方針（丁寧）

```markdown
## コメント

- 関数の目的を簡潔に記述する
- 複雑なロジックには理由を添える
- TODO / FIXME には理由と対応方針を書く
```
