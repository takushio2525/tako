tako の各バージョンの変更内容です。技術的な詳細を含む原文は [GitHub Releases](https://github.com/takushio2525/tako/releases) および [CHANGELOG.md](https://github.com/takushio2525/tako/blob/main/CHANGELOG.md) で確認できます。

:::tip[アップデート方法]
ステータスバーの更新通知をクリックするか、`tako update apply` で最新版に更新できます。詳しくは[セットアップ](/getting-started/#アップデート)を参照してください。
:::

## v0.2.8（2026-07-05）— 最新

### 変更

- **スマホからのリモート閲覧 UI を再設計**（#63）: スマホで tako の画面を見るリモート機能を、1 本の連続スクロールで読める「リーダービュー」に刷新しました。行が画面幅で折り返されるためスマホでも読みやすく、上にスクロールすると過去の出力、最下部に戻ると自動追従を再開します。また、スマホから接続しても PC 側のペインサイズに一切影響しなくなりました

### 修正

- **日本語混在の行で半角文字がまれに消える問題を根治**（#64）: 「ターミナルUI」の末尾の「I」だけ消える、といった描画の不具合を修正しました
- **オーケストレーターのモデル設定が起動のたびに消える問題を修正**（#67): `tako orchestrator profiles set --model` で設定したモデルが `tako master` 実行のたびに削除されていました
- **更新チェッカーの誤報告を修正**（#59）: GitHub の通信制限（レート制限）に当たったとき「更新なし」と誤って表示していた問題を修正し、エラー内容が表示されるようになりました

## v0.2.7（2026-07-03）

### 修正

- **リリースビルドにスマホ用画面（PWA）のビルド工程を組み込み**（#60）: 古い画面が配布物に混入しないよう、ビルド時の検証も追加しました

## v0.2.6（2026-07-03）

### 追加

- **スマホリモート画面の二層構成刷新**（#42, #26）: 過去の出力を自由にスクロール・選択できる履歴表示と、リアルタイム更新されるライブ画面の 2 層構成になりました。Shift+Enter での改行入力にも対応
- **Homebrew 更新失敗からの復旧機能**（#50）: `brew upgrade` が途中で失敗して更新できなくなった状態を自動検知し、`tako update repair` / `tako update apply-zip` で復旧できるようになりました

### 修正

- **macOS の許可ダイアログが更新のたびに出直す問題を修正**（#54）: アプリの署名方法を見直し、アップデートしても一度許可した権限が維持されるようになりました（0.2.5 以前からの更新時のみ、もう一度だけ許可が必要です）
- **スマホのソフトキーボードから Enter が送信できない問題を修正**（#41, #51, #52）
- **全角文字の行をクリックしたときカーソル位置がずれる問題を修正**（#37）
- **負荷が高いときに画面描画が固まることがある問題を修正**（#39）: 描画方式の見直しで描画要素数を 60〜90% 削減
- **日本語入力の変換候補ウィンドウが画面左下に出ることがある問題を修正**（#29）

## v0.2.5（2026-07-03）

### 修正

- **tmux が入っていない環境でも Claude Code の Shift+Enter 改行が効くように修正**（#28）: Homebrew でインストールした標準環境で、複数行の入力ができない問題でした
- **tmux が入っていなくてもタブ構成が保存・復元されるように修正**（#30）: 従来は tmux 無しの環境で再起動するとタブが全て消えていました。tmux 無しでも構成と作業ディレクトリが復元されます（実行中プロセスの完全復元には tmux が必要）
- **裏側のプロセスが不意に終了してもタブ構成が失われないように修正**（#30）

### 追加

- **アプリ内アップデート機能**（#36）: Homebrew / ZIP どちらでインストールしたかを自動判別し、確認ダイアログ → 更新 → 自動再起動まで一気に行います。`tako update status/check/apply` コマンドも追加

## v0.2.4（2026-07-02）

### 修正

- **緊急修正: Pro プランで `tako master` が起動できない問題を修正**（#27）: 既定設定が Max プラン限定のモデルを指定していたため起動に失敗していました。新しい既定は「モデル無指定」（Claude Code の既定モデルで起動）となり、どのプランでも動作します。旧設定は起動時に自動移行されます
- **AI へのプロンプト送達を検証付きに**（#32）: 長い指示文が入力欄に残ったまま送信されない・Enter が空振りする問題を根治しました

### 追加

- **プロファイル管理コマンド**（#27）: `tako orchestrator profiles list/show/set` で、オーケストレーターのモデル設定をコマンドから変更できるようになりました
- **リモートアクセスの基盤強化**（#23）: WebSocket による画面のリアルタイム配信、エージェント一覧 API、会話ログ APIなどを追加

## v0.1.0（2026-06-26）— 初回リリース

tako の最初のリリースです。主な内容:

- **ターミナル基盤**: タブ・ペイン分割・256 色 / truecolor・選択コピー・日本語入力（IME）のインライン変換
- **CLI と内蔵 MCP サーバー**: `tako` コマンドによる画面操作と、Claude Code からのゼロ設定連携（`tako setup-mcp`）
- **パッシブ検知**: dev サーバーのポート検知チップ、タブ・ペインの AI 自動リネーム、コマンド状態の追跡
- **ワークスペース機能**: ファイルツリー、シンタックスハイライト付きコードプレビュー、Markdown プレビュー、git graph + diff ビューア
- **tmux バックエンド永続化**: 再起動しても実行中プロセスと画面内容を完全復元
- **たまり場**: プロセスを生かしたままペインを画面から退避
- **オーケストレーター**: `tako master` によるマルチエージェント連携
- **リモートアクセス**: QR コードで接続できるスマホ用リモート画面
