tako では「1 タブ = 1 作業グループ」で、エージェント・dev サーバー・ログなどを整理します。

## タブ

タブは作業グループの単位です。たとえば「フロントエンド開発」「API サーバー」のようにプロジェクトや作業ごとにタブを分けて使います。

| 操作 | ショートカット |
|---|---|
| 新規タブ | <kbd>Cmd</kbd>+<kbd>T</kbd> |
| タブを閉じる | <kbd>Cmd</kbd>+<kbd>W</kbd> |
| タブ切替 | <kbd>Cmd</kbd>+<kbd>1</kbd>〜<kbd>9</kbd> |
| 次/前のタブ | <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>[</kbd> / <kbd>]</kbd> |

## ペイン分割

1 つのタブ内を分割して、複数のターミナルを並べて表示できます。

| 操作 | ショートカット |
|---|---|
| 右に分割 | <kbd>Cmd</kbd>+<kbd>D</kbd> |
| 下に分割 | <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>D</kbd> |
| ペインを閉じる | <kbd>Cmd</kbd>+<kbd>W</kbd>（最後のペインならタブごと閉じる） |
| フォーカス移動 | <kbd>Cmd</kbd>+<kbd>Alt</kbd>+<kbd>矢印</kbd> |
| リサイズ | <kbd>Ctrl</kbd>+<kbd>Cmd</kbd>+<kbd>矢印</kbd> |

分割はネスト可能です。右に分割したペインをさらに下に分割する、といった柔軟なレイアウトが組めます。

## マウス操作

- **境界線のドラッグ**: ペイン間の境界線をドラッグしてサイズを調整
- **ペインのドラッグ&ドロップ**: タイトルバーをドラッグして、ペインの位置を入れ替え。ドロップ先のハイライトと挿入プレビューが表示されます
- **タブバーのドラッグ**: タブの並び順を変更

## AI からの操作

AI（Claude Code など）は MCP ツールや CLI を通じて、人間と同じレイアウト操作ができます。

```bash
# 右に分割して新しいペインでコマンド実行
tako split --right -- npm run dev

# レイアウトを均等化
tako equalize

# 現在のレイアウトを確認
tako list
```

AI がペインを分割すると、新しいペインは必ず**呼び出し元と同じタブ**に生成されます。作業グループが意図せず散らばることはありません。

## タブの最小化

タブバーの「ー」ボタンを押すと、タブ内の全ペインを「たまり場」に退避できます。処理は動いたまま画面からは消えるので、作業中のペインを一時的に片付けるのに便利です。

詳しくは [たまり場](/features/shelving/) を参照してください。
